# 电商平台流量异常分析与优化建议生成系统 - 使用说明书

## 1. 项目简介

### 1.1 项目概述
电商平台流量异常分析与优化建议生成系统是一个基于人工智能和机器学习技术的智能分析平台。该系统专为电商平台内容运营团队设计，通过自动化的数据分析流程，帮助运营人员识别高效内容策略，提升用户参与度和流量转化。

### 1.2 核心功能
- **智能数据解析**：用户通过上传Excel文件，系统自动解析包含文章标题、发布时间、流量数据等关键信息
- **异常流量检测**：运用统计学方法和机器学习算法，精准识别表现异常好或异常差的内容
- **内容智能抓取**：自动抓取推文链接内容，进行深度文本分析
- **个性化优化建议**：基于数据分析结果，为每篇内容生成具体可操作的优化建议，包括标题优化、内容改进和发布时间优化

### 1.3 应用价值
- 提升内容策略的数据驱动决策能力
- 识别高效内容模式，复制成功经验
- 优化低效内容，提高整体运营效率
- 增强用户参与度和流量转化率

## 3. 系统功能

### 3.1 数据输入与解析模块

#### 3.1.1 Excel文件格式支持
系统支持标准的Excel格式文件（.xlsx/.xls），要求包含以下数据列：
- **基础信息**：data_id（文章ID）、标题、品牌、发文时间、发文链接
- **内容属性**：内容形式、发文类型
- **流量指标**：7天/14天阅读量、互动量、分享量、好物访问量

#### 3.1.2 数据处理流程
1. **文件验证**：检查文件格式、大小限制（50MB以内）
2. **数据解析**：使用Apache POI解析Excel内容，支持多种日期格式
3. **数据清洗**：处理缺失值、异常值，进行数据类型转换
4. **格式标准化**：统一时间格式、数值格式，确保数据一致性

### 3.2 异常检测模块

#### 3.2.1 统计学方法
- **Z-Score检测**：计算各指标的标准分数，阈值设定为|Z| > 2.0
- **IQR方法**：使用四分位距检测离群值，识别超出Q1-1.5×IQR或Q3+1.5×IQR的数据点

#### 3.2.2 综合评分算法
系统采用加权评分机制：
- 分享量权重：20（最高优先级）
- 互动量权重：10（中等优先级）  
- 阅读量权重：1（基础权重）

#### 3.2.3 异常分类
- **异常好（GOOD_ANOMALY）**：各项指标显著高于平均水平
- **异常差（BAD_ANOMALY）**：各项指标显著低于平均水平
- **正常（NORMAL）**：指标在正常范围内

### 3.3 推文抓取与内容分析模块

#### 3.3.1 内容抓取技术
- **异步抓取**：使用Spring WebFlux实现并发内容获取
- **智能解析**：JSoup多选择器策略，提取主要内容区域
- **内容清洗**：移除广告、导航等无关元素，保留核心文本

#### 3.3.2 内容质量分析
- **关键词分析**：识别正面词汇（推荐、必买、优质等）和互动词汇
- **结构分析**：评估标题长度、段落结构、内容丰富度
- **质量评分**：基于多维度指标计算内容质量分数（0-100分）

### 3.4 优化建议生成模块

#### 3.4.1 标题优化策略
- **长度优化**：建议标题长度控制在15-25字
- **关键词建议**：推荐使用吸引性词汇和数字元素
- **结构优化**：建议使用疑问句式增加互动性

#### 3.4.2 内容优化建议
- **互动性提升**：建议添加提问、征集意见等互动元素
- **视觉优化**：推荐增加图片、视频等多媒体内容
- **信息密度**：建议内容长度和信息丰富度的平衡

#### 3.4.3 发布策略优化
- **时间建议**：基于历史数据分析最佳发布时间窗口
- **平台策略**：建议多平台同步发布扩大影响力
- **系列内容**：为优秀内容建议制作系列延续热度

### 3.5 前端展示模块

#### 3.5.1 数据可视化
- **统计仪表板**：展示总体数据概览和异常分布
- **交互式图表**：使用ECharts展示饼图、柱状图等多种图表类型
- **实时更新**：支持数据刷新和动态图表更新

#### 3.5.2 用户交互功能
- **文件上传**：拖拽式上传界面，支持进度显示
- **数据筛选**：按异常状态、品牌等维度筛选数据
- **详情查看**：弹窗展示文章详细信息和完整建议

### 4.2 数据流程图

```
Excel文件上传 → 数据解析 → 异常检测 → 内容抓取 → 质量分析 → 建议生成 → 结果展示
     │            │         │         │         │         │         │
```

### 4.3 模块协作关系

1. **数据解析模块**：接收用户上传的Excel文件，解析并存储到数据库
2. **异常检测模块**：从数据库读取流量数据，应用算法标记异常状态
3. **内容抓取模块**：异步获取文章链接内容，更新数据库记录
4. **分析建议模块**：综合流量数据和内容信息，生成优化建议
5. **前端展示模块**：通过API获取分析结果，提供交互式界面

## 5. 性能要求与优化

### 5.1 响应时间指标
- **文件上传**：< 5秒（50MB以内文件）
- **数据解析**：< 10秒（1000条记录以内）
- **异常检测**：< 3秒（统计计算）
- **内容抓取**：< 30秒（并发处理，取决于网络状况）
- **结果展示**：< 2秒（数据库查询和渲染）

### 5.2 可扩展性设计
- **水平扩展**：支持多实例部署，负载均衡
- **数据库优化**：支持从H2切换到MySQL/PostgreSQL
- **缓存机制**：可集成Redis提升查询性能
- **异步处理**：内容抓取采用线程池，支持并发控制

### 5.3 优化潜力
- **算法优化**：集成更多机器学习算法（Isolation Forest、LOF等）
- **NLP增强**：集成BERT、GPT等预训练模型
- **实时处理**：支持流式数据处理和实时分析
- **智能推荐**：基于历史数据的智能优化建议

## 7. 安装与配置

### 7.1 环境要求
- **Java**: JDK 17或更高版本
- **Node.js**: 16.0或更高版本
- **Maven**: 3.6或更高版本
- **操作系统**: Windows 10/11, macOS, Linux

### 7.2 快速安装步骤

#### 7.2.1 获取项目代码
```bash
git clone [项目仓库地址]
cd 电商平台流量异常分析系统
```

#### 7.2.2 后端启动
```bash
# 方式1：使用启动脚本（Windows）
start-backend.bat

# 方式2：手动启动
cd BackEnd
./mvnw spring-boot:run
```

#### 7.2.3 前端启动
```bash
# 方式1：使用启动脚本（Windows）
start-frontend.bat

# 方式2：手动启动
cd FrontEnd
npm install
npm run dev
```

### 7.3 配置说明

#### 7.3.1 数据库配置
默认使用H2文件数据库，数据保存在`./data/analysis_db.mv.db`

如需使用MySQL，修改`application.properties`：
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/analysis_db
spring.datasource.username=root
spring.datasource.password=your_password
```

#### 7.3.2 系统参数配置
- **文件大小限制**：默认50MB，可在配置文件中调整
- **线程池大小**：默认10个线程用于内容抓取
- **超时设置**：网页抓取超时10秒

### 7.4 验证安装
1. 后端服务：访问 http://localhost:8080/api/analysis/statistics
2. 前端界面：访问 http://localhost:3000
3. 数据库控制台：访问 http://localhost:8080/h2-console

## 8. 使用流程

### 8.1 数据准备
1. 准备符合格式要求的Excel文件
2. 确保文件包含必要的数据列
3. 检查数据质量，处理明显的错误值

### 8.2 系统操作
1. **上传数据**：在首页上传Excel文件，等待解析完成
2. **查看概览**：进入数据分析页面，查看整体统计
3. **异常分析**：进入异常检测页面，重点关注异常文章
4. **优化实施**：根据系统建议，制定内容优化策略

### 8.3 结果应用
1. **成功复制**：分析异常好的文章，提取成功要素
2. **问题改进**：针对异常差的文章，实施优化建议
3. **策略调整**：基于分析结果，调整整体内容策略
